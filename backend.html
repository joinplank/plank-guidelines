01

Architecture

Simple, consistent layered structure over strict Clean Architecture. Each module owns its own slice — no cross-cutting concerns.

Module structure
application/
  <module>/
    <module>.router.ts        # route definitions
    <module>.controller.ts    # request/response handling
    <module>.service.ts       # business logic
    <module>.model.ts         # types / schemas
    <module>.repository.ts    # DB access for this module
infrastructure/
  tools/                      # agent tools (io, search, fetch…)
  database/                   # shared DB client / base repository
  providers/                  # external services (S3, Resend, Sentry, Auth0…)
shared/
  utils/                      # pure helpers (date, string…)
  config/                     # DI container and app config
  • —Business logic lives in the service layer — controllers only parse input and format output.
  • —Repositories abstract all DB access; services never call Prisma directly.
  • —Infrastructure providers are injected, not imported directly in services.
  • —Keep modules self-contained; cross-module calls go service → service, never repository → repository.
02

Prisma / Data

Typed access, version-controlled migrations, and transactional integrity on every multi-step write.

  • —Always use the typed generated Prisma client — never raw SQL unless for performance with profiled evidence.
  • —Migrations in version control; never run db push in production.
  • —Transactions for multi-step writes; never leave partial state on failure.
03

Code Quality

Strict types everywhere — no escape hatches without justification.

  • —Strict TypeScript: noImplicitAny, strictNullChecks. No any without a comment explaining why.
  • —Python: type hints everywhere, Pydantic for validation at boundaries.
  • —No business logic in route handlers — delegate to the service layer.
04

APIs

Validate at the boundary, fail consistently, version from day one.

  • —Validate all inputs at the boundary (Zod for TS, Pydantic for Python) before entering services.
  • —Consistent error envelope: { error: { code, message, details } }.
  • —Version APIs from day one (/v1/); never break existing contracts.
  • —Rate-limit and auth middleware applied at router level, not inside handlers.
