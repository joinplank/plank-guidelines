App Layers

Three layers: Application for business modules, Infrastructure for external integrations, and Shared for utilities and DI config. Click any layer to explore its structure.

application/

Application

Business modules — each owns its full vertical slice
+

Every module follows the same 5-file anatomy. Add a new module by copying the pattern — no framework registration needed beyond DI.

agent/ (example module)
agent.router.ts— route definitions
agent.controller.ts— req/res only
agent.service.ts— business logic
agent.model.ts— types / schemas
agent.repository.ts— DB access

infrastructure/

Infrastructure

Tools, database abstraction, and external service providers
+

Nothing in the application layer imports from here directly — providers are injected. The database repository is the only shared DB primitive.

Shared

Pure utilities and DI container config
+

Only pure functions and config live here — no business logic, no imports from application or infrastructure layers.

  • —Business logic lives in the service layer — controllers only parse input and format output.
  • —Repositories abstract all DB access; services never call Prisma directly.
  • —Infrastructure providers are injected via Awilix, not imported directly in services.
  • —Keep modules self-contained; cross-module calls go service → service, never repository → repository.
  • —shared/ contains only pure functions and config — no business logic, no framework imports.
