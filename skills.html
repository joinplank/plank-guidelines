01

How skills work

Skills are reusable instruction sets that Claude Code loads on demand. They live in skills-lock.json (third-party) or .claude/skills/ (local) and are invoked with a slash command or triggered automatically by Claude when the task matches.

Invocation

Three ways to activate a skill

1. Slash command — type /skill-name at the start of a message to load the skill explicitly.

2. Auto-trigger — Claude detects matching keywords or file patterns and loads the skill automatically before responding.

3. Scoped skills — skills prefixed with a directory (e.g. plank-guidelines:langchain-fundamentals) activate only when the changed files are under that directory. The most specific match wins.

02

Plank built-in skills

These skills ship with Claude Code and encode Plank's own guidelines. They auto-trigger when you work in a Plank project and are the primary way Plank conventions are enforced during code generation.

/plank-backend Plank

Applies Plank backend guidelines: TypeScript, Python, Fastify, FastAPI, Prisma, and Clean Architecture. Auto-triggers on backend service files.

Triggers on: backend routes, services, repositories, Prisma schema edits

/plank-frontend Plank

Applies Plank frontend guidelines: React, Next.js, Shadcn/ui, Tailwind, TanStack Query, accessibility rules, and component patterns.

Triggers on: React components, pages, hooks, Tailwind config edits

/plank-ai Plank

Applies Plank AI/LLM guidelines: prompts as code, typed LangGraph state, cost attribution, and safety guardrails at output boundaries.

Triggers on: LLM calls, agent definitions, prompt files, LangGraph graphs

/plank-infra Plank

Applies Plank infra/DevOps guidelines: Terraform, Kubernetes, Helm, AWS, GCP, Sentry, and security hardening rules.

Triggers on: Terraform files, Helm charts, Kubernetes manifests, Dockerfiles

/plank-testing Plank

Applies Plank testing guidelines: 70/20/10 pyramid, real DB integrations, MSW mocks, Playwright E2E, and LLM eval pipelines.

Triggers on: test files, spec files, Playwright config

03

AI / Agent building

Skills for building AI-powered features with LangChain, LangGraph, and Deep Agents. Start with /ecosystem-primer before writing any agent code.

/ecosystem-primer AI

Required first read for any agent project. Covers framework selection (LangChain vs LangGraph vs Deep Agents), install, environment setup, and which skill to load next.

Invoke first before any agent building skill

/langchain-fundamentals AI

Create LangChain agents with create_agent, define tools, and use middleware for human-in-the-loop and error handling.

Triggers on: LangChain agent definitions, tool registrations

/langchain-dependencies AI

Package versions, installation, and dependency management for LangChain, LangGraph, LangSmith, and Deep Agents in Python and TypeScript.

Use when: setting up a new project or asking about package versions

/langchain-middleware AI

Human-in-the-loop approval flows, custom middleware hooks, Command resume patterns, and structured output with Pydantic/Zod.

Use when: adding HITL approval or custom middleware to a LangChain agent

/langchain-rag AI

RAG system building: document loaders, text splitters, embeddings (OpenAI), and vector stores (Chroma, FAISS, Pinecone).

Use when: building any retrieval-augmented generation feature

/langgraph-fundamentals AI

Core LangGraph: StateGraph, state schemas, nodes, edges, Command, Send, invoke, streaming, and error handling.

Triggers on: any LangGraph code

/langgraph-cli AI

LangGraph CLI: langgraph new, dev, build, up, deploy, and langgraph.json configuration.

Use when: scaffolding or deploying a LangGraph project

/langgraph-human-in-the-loop AI

Pause-for-approval patterns: interrupt(), Command(resume=…), approval/validation workflows, and the 4-tier error handling strategy.

Use when: adding interrupts, approval gates, or error recovery to a LangGraph graph

/langgraph-persistence AI

State persistence: checkpointers, thread_id, time travel, Store API, and subgraph checkpointer scoping.

Use when: adding memory or conversation history to a LangGraph agent

/deep-agents-core AI

Deep Agents fundamentals: create_deep_agent(), harness architecture, SKILL.md format, and configuration options.

Use when: building any Deep Agents application

/deep-agents-memory AI

Deep Agents persistence: StateBackend (ephemeral), StoreBackend (persistent), FilesystemMiddleware, and CompositeBackend for routing.

Use when: a Deep Agent needs memory, state, or file access

/deep-agents-orchestration AI

Multi-agent orchestration: SubAgentMiddleware, TodoList planning, and human-in-the-loop interrupts within Deep Agents.

Use when: building multi-agent pipelines or adding HITL to a Deep Agent

/managed-deep-agents AI

Deploy and operate Managed Deep Agents in LangSmith: deepagents-cli, Python/TypeScript SDKs, React useStream, REST fallbacks, MCP tools, and interrupts.

Use when: deploying a Deep Agent to LangSmith managed infrastructure

/swarm AI

Fan out many independent items in parallel using a table-driven pattern: one row = one unit of work dispatched to a subagent. Aggregate results automatically.

Use when: processing a batch of independent items concurrently

07

Code quality & workflow

Skills for code quality practices, development workflows, and productivity.

/tdd Quality

Test-driven development workflow: red → green → refactor cycle, naming conventions, and test structure best practices.

Use when: writing new features test-first or learning TDD patterns

/clean-code Quality

Robert C. Martin's Clean Code principles: naming, function size, class structure, error handling, and code readability rules.

Use when: writing, reviewing, or refactoring any code

/programming-organizer Quality

Spec-driven orchestrator. Reads specs/, then runs CodeAgent → TestAgent → ReviewAgent in sequence and writes an HTML report to reports/.

Trigger: say run specs or type /programming-organizer [spec-name]

/caveman Quality

Ultra-compressed communication mode. Cuts token usage ~75% by speaking tersely while keeping full technical accuracy. Levels: lite, full, ultra.

Use when: you want shorter responses or need to conserve tokens
