01

Prompts

Prompts are first-class code artifacts — version-controlled, reviewed, and tested like any other module.

  • —Prompts are code — version-controlled, tested, reviewed like any other artifact.
  • —System prompt separated from user prompt; never interpolate user content into the system prompt.
  • —Input/output schemas validated with Pydantic or Zod before and after every LLM call.
  • —Max tokens and temperature set explicitly — never rely on provider defaults.
02

LangGraph Agents

Typed state as single source of truth. Graph topology encodes the workflow — not branching inside nodes.

  • —State is the single source of truth — agents communicate through typed state, not side channels.
  • —Each node has one responsibility; graph topology encodes the workflow, not branching inside nodes.
  • —Human-in-the-loop checkpoints at irreversible steps (sends, writes, charges).
  • —Retry with exponential backoff on provider errors; surface failures to state, never swallow silently.
03

Cost & Reliability

Log every token, abstract every provider, cache deterministic calls, and stream everything user-facing.

  • —Log token usage per call — aggregate in the observability stack for cost attribution per feature.
  • —Provider abstraction layer: swap Anthropic/OpenAI/Gemini without touching business logic.
  • —Cache deterministic LLM responses (e.g. classification, extraction) when input is stable.
  • —Streaming responses for any user-facing generation; never block on full completion.
04

Safety

Validate before tools, audit every action, guard outputs for PII, and load-test under real concurrency.

  • —Never pass raw user input as tool arguments — validate and sanitize before tool execution.
  • —Audit log every agentic action with inputs, outputs, and actor identity.
  • —Guardrails at output boundary: classify/redact PII before returning to users.
  • —Load test prompt pipelines under concurrency; LLM latency collapses under load differently than APIs.
