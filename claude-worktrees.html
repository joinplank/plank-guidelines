01

Quick Start

Each claude --worktree <name> creates a fresh git checkout under .claude/worktrees/<name>/ on a new branch — edits in one session never touch files in another.

# Accept workspace trust once (required before first --worktree use)
claude

# Parallel session on a new branch
claude --worktree feature-auth

# Second session in another terminal
claude --worktree bugfix-123

# Branch from a specific PR
claude --worktree "#1234"

# Auto-generated name (e.g. bright-running-fox)
claude --worktree
  • —Add .claude/worktrees/ to .gitignore so worktree contents don't appear as untracked files in the main checkout.
  • —By default worktrees branch from origin/HEAD (clean remote state). Set worktree.baseRef: "head" in settings to branch from local HEAD instead.
  • —After creating a worktree, install dependencies in it before running code (npm install, pip install, etc.).

Workflow Patterns

Four proven patterns — choose by how much coordination you need between sessions.

Parallel Feature Development

One terminal per worktree, each on its own branch. Zero interference — every session edits its own files.

Terminal A: claude --worktree feature-payments
Terminal B: claude --worktree feature-notifications
Terminal C: claude --worktree bugfix-auth-timeout

Asymmetric (you + one agent)

You edit in the main checkout; one agent works autonomously in a worktree. Neither waits on the other.

Main terminal:  # code normally in repo root
Terminal B:     claude --worktree refactor-dal   # runs unattended

Research → Implementation Pipeline

Three overlapping phases, each isolated. Start implementation as soon as the spec is stable enough — phases can run concurrently.

WorktreeRoleTask
researchExplorerReads codebase, produces SPEC.md
implBuilderReads SPEC.md, writes code
main checkoutReviewerMerges, reviews diff, opens PR

Writer / Reviewer

Two sessions for unbiased review — the reviewer never wrote the code, so it evaluates the result independently.

Session A: claude --worktree impl-rate-limiter
           → "implement rate limiting for our API"

Session B: claude --worktree review-rate-limiter
           → "review @src/middleware/rateLimiter.ts for edge cases,
              race conditions, and consistency with existing middleware"

Session A: "address this review feedback: [paste Session B output]"

Copy Local Config into Worktrees

Worktrees are fresh checkouts — .env and similar gitignored files are absent. Use .worktreeinclude to copy them automatically.

.worktreeinclude (gitignore syntax)
.env
.env.local
config/secrets.json
  • —Only files that match and are gitignored are copied — tracked files are never duplicated.
  • —Applies to --worktree sessions, subagent worktrees, and desktop parallel sessions.
05

Subagent Isolation

Subagents can each run in their own worktree so parallel edits don't conflict. Set it per-agent or ask Claude at runtime.

.claude/agents/security-reviewer.md
---
name: security-reviewer
description: Reviews code for security vulnerabilities
tools: Read, Grep, Glob, Bash
isolation: worktree       # each subagent gets its own worktree
---
  • —Subagent worktrees are removed automatically when the subagent finishes with no uncommitted changes.
  • —Subagent worktrees branch from the same base as --worktree (respects worktree.baseRef setting).
  • —Ask Claude at runtime: "use worktrees for your agents" instead of setting frontmatter.
06

Cleanup

Cleanup behavior depends on whether the session made any changes. Non-interactive runs never auto-clean.

No changes

Worktree and branch removed automatically on exit.

auto cleanup

Uncommitted changes or new commits

Claude prompts: keep (preserves directory + branch) or remove (deletes both, discards commits).

prompted

Non-interactive (-p flag)

No cleanup prompt. Remove manually with git worktree remove.

manual
git worktree list                           # inspect all worktrees
git worktree remove .claude/worktrees/foo   # manual removal
git worktree remove --force ...             # discard uncommitted changes
07

Context Management

Context window is the most important resource. Keep each session focused — clear between unrelated tasks.

  • —Use /clear between unrelated tasks within a session — long context degrades performance.
  • —Use claude --continue to resume the most recent session; claude --resume to pick from a list.
  • —Use /compact or Esc+Esc → Summarize when context fills mid-task.
  • —Delegate exploratory reads to a subagent so main session context stays clean.
  • —Always inspect the full diff before merging: git diff main..worktree-feature-auth — catch scope creep at diff time, not in prod.
