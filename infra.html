🏗

Infrastructure as Code

Every resource declared in Terraform. No click-ops in shared environments, ever.

🔒

Security by default

Least-privilege IAM, network default-deny, distroless base images, and CVE scanning in CI.

📡

Full observability

Structured logs, distributed traces via OTel, Sentry for errors, and SLO-based alerting.

⚙️

Pod reliability

Resources & limits, liveness/readiness probes, and RBAC least-privilege on every workload.

01

Terraform

Remote state, modular structure, and secrets via secret managers — never in .tfvars.

  • —Remote state in S3/GCS with state locking — never local state in shared environments.
  • —One root module per environment; shared infra in reusable child modules.
  • —All secrets via data.aws_secretsmanager_secret or GCP Secret Manager — never in .tfvars.
  • —Plan output reviewed before apply in CI; auto-apply only for non-prod.
02

Kubernetes / Helm

Every pod gets resource limits, probes, and a least-privilege ServiceAccount. No exceptions for "quick deploys".

  • —Resources and limits on every container. No BestEffort QoS in production.
  • —Helm values per environment (values-prod.yaml, values-staging.yaml) — no env-specific logic inside templates.
  • —Liveness and readiness probes on every workload. Never skip for "quick deploys".
  • —RBAC: least-privilege ServiceAccounts per workload. No default SA for pods touching the API.
03

Observability

Structured logs everywhere, errors in Sentry, traces propagated end-to-end, alerts on SLOs not raw metrics.

  • —Sentry for error capture: environment tags, release tracking, and source maps uploaded on every deploy.
  • —Structured logs (JSON) everywhere — no plain string logging in production code.
  • —Alerts on SLOs, not just raw metrics. Define error budget before shipping a service.
  • —Traces propagated through all service boundaries via OTel context headers.
04

Security

Non-root images, default-deny network policies, workload identity, and CVE blocking in CI.

  • —Images: non-root user, read-only root filesystem, distroless or minimal base.
  • —Network policies default-deny in each namespace; allow only required paths explicitly.
  • —IAM: workload identity / IRSA for AWS/GCP — no long-lived key files in clusters.
  • —Vulnerability scanning in CI (Trivy or equivalent) — block on critical CVEs.
