Client architecture always wins

When working inside a client's project, follow their existing architecture, conventions, and tooling decisions — even when they differ from this document. These guidelines are a starting point, not a mandate. Client context overrides everything here.

2

For new Plank projects & good practice

These guidelines drive new projects started by the Plank team and serve as a reference for recommended practices when advising clients. Use them to bootstrap a project with consistent, well-reasoned defaults — then adapt to the specific context.

Spec-driven development

Write the spec, Claude does the rest

Create specs/<name>_<YYYY-MM-DD>.md with goal, acceptance criteria, constraints, and context. Say run specs and the programming-organizer skill orchestrates the full pipeline: CodeAgent implements, TestAgent verifies, ReviewAgent audits — you get an HTML report. Run it inside a worktree so you can ship to a PR directly without touching your main branch.
